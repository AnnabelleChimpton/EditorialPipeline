# Pipeline Examples

These are real outputs from a running instance of this pipeline: the **Vodka Cranberry Era** project, which covers millennial internet nostalgia.

The examples follow one topic — **MySpace profile culture and the tyranny of the Top 8** (ID: `vce-001`) — as it moves through each pipeline stage. This lets you see how raw research becomes a finished, review-ready post.

## What's here

| File | Stage | What it is | Source |
|------|-------|------------|--------|
| `01-card.md` | Stage 2 | First-pass research card | Live production output |
| `02-dossier.md` | Stage 3 | Deep-dive dossier | Live production output |
| `03-brief.md` | Stage 4 | Post brief (format + creative direction) | Illustrative example |
| `04-draft.md` | Stage 5 | Post draft (ready-to-review copy) | Illustrative example |
| `05-qa-report.md` | Stage 6 | QA report (self-edit before human review) | Illustrative example |
| `05b-revision.md` | Stage 7 | Revised draft addressing QA feedback | Illustrative example |

**Stages 1-3** (card and dossier) are actual production output from automated cron runs. What you see is exactly what the pipeline produced.

**Stages 4-7** (brief, draft, QA report, revision) are illustrative examples showing the expected format and quality bar. They demonstrate what the pipeline targets when it converts research into publishable content. The revision example (`05b`) shows what a revision would look like if QA had flagged an issue — it's a hypothetical "what if QA said needs-edits" scenario, since the QA report in `05-qa-report.md` actually approved the draft.

Stage 1 (sourcing) is not shown here because it produces index entries rather than standalone files.
